1) Model the state around the data

UI state should match real domain objects. Avoid many small flags that can conflict.

2) Loading and error states

Always design for loading, empty, and error. It prevents hacks later.

3) One source of truth

Keep core data in one place. Derive views from it. This keeps the UI predictable.

4) Ship small changes

Reduce UI risk by shipping small. Add instrumentation so you can see what users do.
